# 技术设计

## 一、技术栈

本作品集网站采用前端主流轻量化、高性能技术栈，兼顾开发效率、页面质感与后续维护拓展，具体技术选型如下：

- **核心框架**：React \+ TypeScript \+ Vite

- **样式方案**：Tailwind CSS（实现深色主题、响应式布局、极简科技风格）

- **路由管理**：React Router（支持多页面路由切换、页面跳转）

- **动画交互**：Framer Motion（实现流畅滚动动画、模块渐入、卡片hover动效）

技术选型优势：轻量化打包、首屏加载速度快、类型约束严谨、动画效果丝滑，完美适配个人静态作品集网站需求，兼顾美观、性能与可维护性。

## 二、项目整体结构

采用模块化、组件化分层架构，代码解耦清晰，便于后期迭代更新项目、技能内容，整体目录结构如下：

    Footer.tsx       // 底部版权、兜底组件
  data/              // 静态数据文件（统一管理可配置内容）

## 三、数据管理方案

本项目为静态作品集网站，无后端接口依赖，所有展示数据采用**本地TS静态数据管理**，轻量化、易维护、零部署成本。

### 3\.1 数据存储方式

所有动态展示内容统一封装在 `src/data/` 目录下的TS文件中，通过**TypeScript数组\+类型定义**统一管理，规避类型错误，保证数据规范。

### 3\.2 核心数据说明

- **skills\.ts**：存储个人所有技术技能、职业能力标签，包含技能名称、分类、熟练度等字段，支持随时新增、删除、修改技能内容，页面自动渲染更新。

- **projects\.ts**：存储所有机器视觉作品集项目数据，包含项目名称、简介、技术栈、项目截图、项目链接、核心价值等字段，结构化管理所有项目案例，后续新增项目仅需在此文件添加数组对象即可，无需修改页面组件代码。

### 3\.3 数据渲染逻辑

页面组件通过引入对应TS数据文件，遍历数组自动渲染技能列表、项目卡片，实现**数据与视图分离**，大幅降低维护成本，适配长期迭代更新。

## 四、核心功能技术实现要点

### 4\.1 主题与样式实现

基于Tailwind CSS自定义**深色主题全局配色**，配置科技蓝辅助色、深色基底、分层文字色，统一全站视觉风格，适配极简科技风设计要求，同时依托Tailwind原生特性实现移动端、桌面端响应式自适应。

### 4\.2 动画效果实现

通过Framer Motion实现全站交互动画：页面滚动模块渐入、卡片悬浮上浮、按钮hover微动、平滑页面过渡，动画轻量化不卡顿，贴合高端极简作品集质感。

### 4\.3 路由与页面适配

基于React Router实现单页多模块路由切换，支持锚点定位、页面跳转，适配不同浏览场景，保证多端浏览流畅度与一致性。

## 五、拓展与维护说明

- 新增/修改技能、项目内容，仅需修改 `data` 目录下对应数据文件，无需改动组件结构；

- 支持后续新增页面、新增动画效果、拓展社交链接，架构可拓展性强；

- 纯静态项目，可直接部署至Vercel、Netlify、GitHub Pages等平台，部署便捷。
